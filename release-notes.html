Overview

Theonix OS is a modern, Arch-based Linux distribution. Version 1.0 “Genesis” is its first public release: a rolling system that pairs the Arch base and Pacman with a KDE Plasma 6 desktop on Wayland, a custom Calamares installer, its own package repository and consistent Theonix branding throughout. The goal of Genesis is a desktop that feels coherent from first boot — installer, repository, desktop and branding tuned to work together — delivered as a free and open source project.

What's new

Highlights

The headline changes that make up the Genesis release.

Initial public release

Genesis is the first time Theonix OS is available to the public. It brings the whole system together into a single, installable release for x86_64 hardware.

Theonix branding & theme

A new visual identity and theme run across the desktop, giving Theonix a consistent look from the login screen to the applications you use every day.

Custom Calamares installer

A branded Calamares installer handles partitioning, user creation and the bootloader. It takes you from the live session to an installed desktop in a few guided steps.

Custom package repository

Theonix ships its own package repository. It delivers the branding, defaults and tooling that make the system feel complete, alongside the standard Arch packages.

Fastfetch branding

Fastfetch is configured with Theonix branding, so opening a terminal presents a clean, themed summary of your system information out of the box.

KDE Plasma 6 desktop

Genesis is built around KDE Plasma 6, a modern and highly configurable desktop environment that anchors the Theonix experience.

Wayland by default

The Wayland session is enabled by default, providing a modern display stack for the Plasma 6 desktop.

Improved bootloader & GRUB fix

The bootloader configuration has been improved for a more reliable startup, and an issue with GRUB installation has been resolved.

Known considerations

A few practical notes to keep in mind for the Genesis release.

Secure Boot

On some systems you may need to disable Secure Boot in your firmware settings to boot the live media. You can re-enable it later depending on your setup.

NVIDIA graphics

If you use an NVIDIA GPU, the proprietary drivers can be installed after installation. Consult the documentation for the recommended steps on your hardware.

Reporting issues

If you run into a problem, please report it on GitHub. Clear reports help us improve future releases. Open an issue
