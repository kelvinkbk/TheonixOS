Ways to contribute

Every bit helps

You don’t need to write code to make Theonix better. Here are some of the ways people pitch in.

Report bugs

Hit something broken? Open an issue on GitHub with steps to reproduce and it can get fixed.

Test releases

Try new ISOs and updates, then share what worked and what didn’t. Real-world feedback is gold.

Package software

Help package and maintain software for the Theonix repository so more works out of the box.

Improve docs

Fix a typo, clarify a step or write a guide. Good documentation helps everyone who comes next.

Help translate

Speak more than one language? Help make Theonix and its docs accessible to more people.

Help other users

Answer questions on the forum or Discord. A helpful reply goes a long way in a small community.

Code of conduct

Be kind, be welcoming

Theonix aims to be a friendly, respectful place. Treat others the way you’d want to be treated: be patient with newcomers, assume good faith, keep criticism constructive, and leave harassment and discrimination at the door.

Everyone was new once. A welcoming community is what turns a small project into a lasting one — help keep it that way.
