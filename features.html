Features

What actually makes Theonix

No generic checklist. Every piece below ships in Theonix OS today — a rolling Arch base, KDE Plasma 6 on Wayland, a branded Calamares installer and a dedicated package repository — assembled so the system feels coherent from first boot. The one planned feature is clearly marked.

Rolling Arch base

One install, continuous updates

Theonix OS is built directly on Arch Linux and managed with Pacman. There are no fixed releases to reinstall over — you install once and keep receiving the latest kernels, drivers and applications as they land. The Theonix repository sits alongside the standard Arch repositories, so branding and defaults update through the same familiar upgrade command.

  • Native Pacman package management
  • No version jumps, no reinstall churn
  • Custom Theonix repository in the mix
theonix@genesis — konsole
theonix@genesis:~$ sudo pacman -Syu
:: Synchronising package databases…
core extra theonix multilib
✓ System up to date — powered by the Theonix repository.
theonix@genesis:~$ flatpak install flathub org.kde.kdenlive
✓ Installation complete.

KDE Plasma 6 · Wayland

A modern desktop, themed end to end

Theonix ships KDE Plasma 6 running on a Wayland session by default, with XWayland available for legacy X11 applications. The custom Theonix theme, wallpaper, icons and login screen are applied throughout so the desktop looks deliberate the moment you log in — not like a stock install you still have to configure.

The full picture

Everything Theonix brings together

Each of these is part of the shipped system — with one clearly-labelled item still on the roadmap.

Arch Linux base

Built on a clean Arch Linux foundation, giving you the current Arch ecosystem and documentation underneath a curated setup.

Rolling release

Install once and stay current. Updates arrive continuously through Pacman, with no major-version upgrades to reinstall over.

Pacman

The fast, dependency-aware Arch package manager handles system software, with the Theonix repository enabled alongside the defaults.

Flatpak

Flatpak support is available for sandboxed, always-current desktop apps from Flathub — ideal for software outside the Arch repos.

KDE Plasma 6

A modern Plasma 6 desktop is the default environment, dressed in the custom Theonix theme and branding throughout.

Wayland

The Plasma session runs on Wayland by default for a modern display stack, with XWayland covering legacy X11 applications.

Fast boot

A lean systemd service set and a streamlined boot path keep startup light, getting you to a usable desktop quickly.

Custom repository

A dedicated Theonix package repository ships the branding, defaults and tooling — and delivers them as ordinary rolling updates.

Calamares installer

A branded Calamares installer walks you through partitioning, users and the bootloader — from live USB to installed desktop in minutes.

Fastfetch branding

Fastfetch is pre-configured with Theonix branding, so a quick system-info readout shows the distribution off in the terminal.

Theonix branding & theme

A cohesive visual identity — theme, wallpaper, icons and login screen — is applied across the system for a consistent look.

AI assistant Planned

A local, privacy-respecting AI assistant is on the roadmap — on-device help without sending your data to the cloud. Not yet shipped.
