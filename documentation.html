Step by step

Installation overview

Theonix OS ships as a live ISO. You boot the live session, try the desktop, then launch the custom Calamares installer to write it to disk. The whole flow — download, flash, boot, install — takes only a few minutes of hands-on time.

  1. 01

    Download & verify

    Get the ISO from the download page then confirm its SHA256 checksum before flashing — follow the verification guide.

    sha256sum TheonixOS-1.0.iso
  2. 02

    Create a bootable USB

    On Linux, write the image directly to the USB device with dd. Replace /dev/sdX with your actual USB device — this erases everything on it, so double-check with lsblk first.

    sudo dd if=TheonixOS-1.0.iso of=/dev/sdX bs=4M status=progress oflag=sync

    Prefer a graphical tool? Balena Etcher (Linux, macOS, Windows), Rufus (Windows) and GNOME Disks all write the ISO in a few clicks — use DD / image mode when Rufus offers a choice.

  3. 03

    Boot from the USB

    Insert the stick, restart, and open your firmware boot menu to pick the USB device (see Booting the USB). The live Theonix desktop loads without touching your disk.

  4. 04

    Run Calamares

    Launch the Install Theonix OS icon on the live desktop. The branded Calamares installer walks you through language, timezone, keyboard, partitioning (erase disk, install alongside, or manual), and creating your user account and bootloader.

  5. 05

    Reboot

    When the installer finishes, remove the USB stick and reboot. You’ll land on the themed SDDM login screen and into your new Theonix OS desktop — ready to update and make your own.

Firmware

Booting the USB

Most machines don’t boot from USB automatically — you have to tell the firmware to. Right after powering on, tap the boot-menu or setup key repeatedly before the operating system starts loading.

The key varies by vendor. Common ones are F12, F2, Del and Esc; some laptops use F9 or F10. Watch the splash screen for a “Boot Menu” or “Setup” hint.

In the boot menu, select the entry that names your USB stick (often the brand or “USB”). If it appears twice, a UEFI-prefixed entry boots in UEFI mode — the recommended choice on modern hardware.

Firmware modes

BIOS vs UEFI

Theonix OS boots on both legacy BIOS and modern UEFI firmware. UEFI is recommended: it supports large disks with GPT partitioning, faster boot paths and Secure Boot.

Not sure which you have? After installation you can check from a terminal — if this directory exists, you booted in UEFI mode:

ls /sys/firmware/efi && echo UEFI || echo BIOS

Firmware settings often label the mode as UEFI, Legacy, or CSM (Compatibility Support Module). For a fresh install on recent hardware, disable CSM / Legacy and boot the USB in UEFI mode so your installed system matches.

Firmware security

Secure Boot

Secure Boot only allows firmware-signed boot loaders to run. The Theonix live media is not signed with the keys shipped by your firmware vendor, so you may need to disable Secure Boot to boot the installer.

To disable it, enter your firmware setup (see the vendor table above), find Secure Boot under a Security or Boot tab, set it to Disabled, then save and exit. On some boards you must also set an administrator/supervisor password before the option becomes editable.

Re-enabling Secure Boot later is possible but takes extra work: you would need to sign the kernel and boot loader and enrol your own keys (for example with a Machine Owner Key). This is an advanced, optional step — many users simply leave Secure Boot off. Only re-enable it once you understand the signing your setup requires, or an unsigned kernel update can leave the machine unbootable.

Alongside another OS

Dual boot

You can run Theonix OS alongside Windows or another Linux distribution on the same machine and choose between them at boot.

  • •Back up first. Repartitioning always carries some risk — copy anything important off the machine before you start.
  • •Free up space. Inside the existing OS, shrink a partition to leave unallocated room for Theonix (aim for at least the recommended storage).
  • •Install alongside. In Calamares, pick Install alongside and target the free space, or use manual partitioning if you want full control.
  • •GRUB detects the rest. Theonix installs the GRUB bootloader; with os-prober it finds other installed systems and adds them to the boot menu.

If another OS isn’t listed after install, enable detection and regenerate the config:

sudo grub-mkconfig -o /boot/grub/grub.cfg

You may first need to uncomment GRUB_DISABLE_OS_PROBER=false in /etc/default/grub.

Windows clock caveat

Windows assumes the hardware clock is in local time, while Linux defaults to UTC — so dual-boot clocks can drift by your timezone offset. The cleanest fix is to make Linux use local time to match Windows:

timedatectl set-local-rtc 1 --adjust-system-clock

Software

Package management

Theonix OS installs software two complementary ways: Pacman, the native Arch package manager backed by the official repos and the Theonix repository, and Flatpak for sandboxed apps from Flathub.

Pacman

Native packages — run with sudo to make changes.

Install a package

sudo pacman -S name

Search the repositories

pacman -Ss term

Remove a package & its deps

sudo pacman -Rns name

Flatpak

Sandboxed apps from Flathub, great for proprietary or newer builds.

Install an app

flatpak install flathub org.app

Search Flathub

flatpak search app

Run an installed app

flatpak run org.app

You can also browse and install both Pacman and Flatpak software graphically with KDE Discover.

Staying current

Updates

Theonix OS is a rolling release — there are no big version upgrades. You install once and keep the system current with regular updates.

Update the full system (databases and packages) with a single command:

sudo pacman -Syu

Keep your Flatpak apps fresh too:

flatpak update

Update regularly rather than saving up months of changes — smaller, frequent updates are less disruptive. Before a large update, it’s good practice to skim the latest Arch news for any manual interventions, and avoid partial upgrades (never run -Sy on its own to install a single package).

When things break

Recovery

If the desktop won’t start or a boot fails, you can usually repair the system without reinstalling.

Boot to a TTY

If graphics fail but the system boots, switch to a text console with Ctrl+Alt+F3 and log in. From there you can inspect logs, roll back a package or reconfigure the display.

Chroot from the live USB

Boot the Theonix live USB, then mount your installed root (and EFI partition, if UEFI) and enter it with arch-chroot. Substitute your real partition names from lsblk.

sudo mount /dev/sdXn /mnt && sudo arch-chroot /mnt

Reinstall GRUB

From inside the chroot, reinstall the bootloader (UEFI example) and regenerate its config:

grub-install --target=x86_64-efi --efi-directory=/boot/efi --bootloader-id=Theonix
grub-mkconfig -o /boot/grub/grub.cfg

Fix a broken update

If an interrupted update left the system inconsistent, from a TTY or chroot refresh the keyring, then complete the upgrade:

sudo pacman -Sy archlinux-keyring && sudo pacman -Su

If a single recently upgraded package is the culprit, you can roll it back from the pacman cache in /var/cache/pacman/pkg/ with sudo pacman -U.

Common issues

Troubleshooting

A few problems come up often. Expand a topic for the fix.

The USB won’t boot

Work through these in order:

  • •Disable Secure Boot in firmware — unsigned live media is blocked otherwise.
  • •Re-flash the stick using DD / image mode; a file-copy or incomplete write won’t boot.
  • •Verify the checksum — a corrupt download fails silently. Re-download if the SHA256 doesn’t match.
  • •Try another USB port (prefer a rear or USB 2.0 port) or a different stick.
No Wi-Fi after install

Connect from a terminal using NetworkManager’s text UI:

nmtui

Choose Activate a connection, pick your network and enter the password. If no wireless device appears at all, your adapter may need a firmware package (for example linux-firmware) — connect over Ethernet or USB tethering once to install it.

NVIDIA black screen on boot

At the GRUB menu, press e to edit the boot entry, add nomodeset to the linux line, then boot with Ctrl+X. Once you reach the desktop, install the proprietary driver:

sudo pacman -S nvidia

Reboot afterwards. The temporary nomodeset option is only needed to get in the first time.
